Guía de Restauración de Base de Datos

Volver

1. Restauración de Archivos .dump (Nativo de PostgreSQL)

Este es el formato comprimido nativo. Para restaurar este archivo en Railway, asegúrese de tener instalada la herramienta pg_restore de PostgreSQL y ejecute el siguiente comando en su terminal local: 

PGPASSWORD="TU_PASSWORD_AQUI" pg_restore -h TU_HOST_AQUI -p TU_PUERTO_AQUI -U TU_USUARIO_AQUI -d TU_BASE_DATOS_AQUI -c --no-owner --no-privileges NOMBRE_RESPALDO.dump
Nota de comandos adicionales: El parámetro -c (clean) eliminará las tablas existentes antes de restaurarlas para evitar errores de duplicado. --no-owner y --no-privileges evitan errores de asignación de usuarios y permisos incompatibles entre entornos.

2. Restauración de Archivos .json (Django loaddata)

Si el sistema generó un archivo JSON debido a la falta de herramientas de Postgres en el servidor, puede cargarlo directamente a través del motor de base de datos que Django tenga configurado en su archivo .env local ejecutando: 

python manage.py loaddata NOMBRE_RESPALDO.json
¡ATENCIÓN! La restauración de bases de datos sobrescribirá los datos del entorno seleccionado. Le recomendamos realizar siempre una descarga de seguridad manual del estado actual antes de proceder a una restauración.
